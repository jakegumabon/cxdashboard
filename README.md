# CX Analytics Dashboard

Automated pipeline: **Zendesk Explore email → GitHub → Dashboard on GitHub Pages**

## How it works

```
Zendesk Explore (scheduled report)
        │
        ▼
  Gmail (zip with 4 CSVs)
        │
        ▼
  Google Apps Script (every 30 min)
  - finds new email
  - extracts zip attachment
  - commits to data/latest.zip
        │
        ▼
  GitHub Actions (on push to data/)
  - unzips CSVs
  - runs csv_to_dashboard.py
  - injects data into HTML template
  - deploys to GitHub Pages
        │
        ▼
  Live dashboard at:
  https://jakegumabon.github.io/cxdashboard/
```

## Project structure

```
cxanalytics/
├── .github/workflows/

├── scripts/

├── template/
│   └── dashboard.html          # Dashboard HTML with data placeholders
├── google-apps-script/
│   └── Code.gs                 # Apps Script: Gmail → GitHub
├── data/
│   └── latest.zip              # Latest Zendesk export (committed by Apps Script)
├── docs/
│   └── index.html              # Built dashboard (generated by Actions)
└── README.md
```

## Setup (one-time)

### 1. GitHub Repository

1. Push this repo to `github.com/jakegumabon/cxdashboard`
2. Go to **Settings → Pages**
3. Set Source to **GitHub Actions**
4. (Optional) The dashboard URL will be `https://jakegumabon.github.io/cxdashboard/`

### 2. GitHub Personal Access Token

1. Go to **github.com → Settings → Developer settings → Personal access tokens → Fine-grained tokens**
2. Create a new token:
   - Name: `cx-dashboard-automation`
   - Repository access: Only select `jakegumabon/cxdashboard`
   - Permissions: **Contents → Read and write**
3. Copy the token (you'll need it for Apps Script)

### 3. Google Apps Script

1. Go to [script.google.com](https://script.google.com) → **New project**
2. Replace the contents of `Code.gs` with the code from `google-apps-script/Code.gs`
3. Go to **Project Settings** (gear icon):
   - Under **Script Properties**, add:
     - `GITHUB_TOKEN` = your personal access token from step 2
     - `GITHUB_REPO` = `jakegumabon/cxdashboard`
4. Run `setup()` from the editor — this creates the time-based trigger
   - Google will ask for Gmail permissions — approve them
5. Run `testRun()` to verify it works with the latest email

### 4. Zendesk Explore Schedule

Make sure your Zendesk Explore dashboard "For Claude" is scheduled to send daily. The email should contain a zip with these 4 CSVs:
- BI Customer Contacts - Support Requests + Question/Guidance
- BI Customer Contacts - Product Issue
- BI Operational Volume - New tickets
- BI Operational Volume - Reopened tickets

## Manual build (local)

```bash
# From a zip file
python scripts/csv_to_dashboard.py data/latest.zip \
  --template template/dashboard.html \
  --output docs/index.html

# From pre-extracted CSVs
python scripts/csv_to_dashboard.py --csv-dir /path/to/csvs/ \
  --template template/dashboard.html \
  --output docs/index.html
```

## Updating agent-team mapping

Edit `ASSIGNEE_GROUP` in `scripts/csv_to_dashboard.py`. The dashboard will flag any agents not in this mapping.

## Troubleshooting

- **Apps Script not finding emails**: Check the Gmail search query in `CONFIG.GMAIL_QUERY`. Make sure the Zendesk email subject matches.
- **GitHub Actions failing**: Check the Actions tab for build logs. Most likely a CSV format change.
- **Dashboard shows old data**: Check if `data/latest.zip` was updated. Check Apps Script execution log at script.google.com.
